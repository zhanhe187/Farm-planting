# 农场种植管理系统 — 使用说明文档

## 一、系统概述

农场种植管理系统（FPMS）是一个面向中小型农场的种植生产管理平台，覆盖从地块规划、播种、生长管理、采收、库存到销售的全链路。系统采用 Spring Boot + Thymeleaf 架构，数据库为 SQL Server，所有状态值、操作类型、物料分类等关键业务数据均以中文存储，便于农业从业人员直接阅读和使用。

### 核心功能

| 模块 | 功能 |
|---|---|
| 仪表盘 | 地块可视化布局、批次状态统计图表、营收KPI |
| 种植批次 | 批次生命周期管理，状态机控制推进 |
| 农事作业 | 播种/施肥/施药/灌溉/除草/巡检/采收等作业记录 |
| 库存管理 | 农资出入库、安全库存预警、出库流水 |
| 采收与溯源 | 采收记录、自动生成溯源码、公开溯源页面 |
| 销售管理 | 在售农作物购买、销售订单记录、营收统计 |
| AI 助手 | 种植知识问答、作物图片识别（Vision） |
| 设备绑定 | 手机扫码绑定，实现移动端拍照识别 |

### 系统运行逻辑

系统以“种植批次”为主线串联业务：

1. 超级管理员或农场主维护基础资料，包括地块、可种植农作物、农资库存和 AI 端点。
2. 农场主或农技员创建种植批次，选择地块和作物，批次默认进入「已计划」。
3. 农技员推进批次状态，田间工人或农技员记录播种、施肥、施药、灌溉、除草、巡检等农事作业。
4. 作业选择农资时自动扣减库存，并生成出库流水；仓库管理员负责农资建档、采购入库、成本价、安全库存维护，以及在库存页新增种子对应的可种植作物。
5. 批次达到采收条件后，系统先校验农药安全间隔期，再登记采收并生成公开溯源码。
6. 农场主或超级管理员可手工登记销售记录；客户账号可在「购买农作物」中选择在售作物自助购买，成功后系统自动生成本人销售流水。
7. 公开溯源页无需登录，手机扫码可查看脱敏后的批次、采收和农事信息。

## 二、环境部署

### 2.1 准备工作

- JDK 17+
- Maven 3.6+
- SQL Server（本机 localhost:1433）

### 2.2 初始化数据库

使用 SQL Server 命令行工具执行初始化脚本：

```powershell
sqlcmd -S localhost -U sa -P 123456 -C -b -f 65001 -i src\main\resources\db\init-daihaojie-xm.sql
```

该脚本完成以下工作：
- 创建数据库 `daihaojie_xm`
- 建立所有表结构（sys_user、farm_plot、farm_crop、farm_material、plant_batch 等）
- 插入示例数据（6个地块、7种作物、13种农资、6个种植批次、23条农事记录）
- 建立触发器（农事操作不可篡改）和索引视图

### 2.3 启动应用

```bash
mvn spring-boot:run
```

访问 `http://localhost:8080` 进入系统。

- **客户**：可购买在售农作物，查看与自己客户名称匹配的销售流水，以及访问公开溯源页面；不能进入生产、库存和 AI 管理功能。

## 四、完整业务操作流程

### 4.1 流程图

```
地块规划 ──→ 创建批次 ──→ 播种 ──→ 生长期管理 ──→ 待采收 ──→ 采收 ──→ 已完结
                │                        │                      │
                │                        │                      ├──→ 销售出库
                │                        │                      └──→ 生成溯源码
                │                        │
                └──→ 已废弃（疫病/灾害等异常中止）

农事作业贯穿整个生长期：
  播种 → 灌溉 → 施肥 → 施药 → 除草 → 病虫害巡检 → 采收

每次作业消耗农资，自动扣减库存生成出库流水。
```

### 4.2 批次状态说明

批次共有 7 种状态，状态机控制推进路径：

| 状态 | 含义 | 说明 |
|---|---|---|
| 已计划 | PLANNED | 批次已创建，尚未播种 |
| 已播种 | SOWED | 已完成播种作业 |
| 生长期 | GROWING | 作物正在生长，可记录各类农事作业 |
| 待采收 | READY_HARVEST | 作物已达到采收标准 |
| 采收中 | HARVESTING | 采收进行中（短暂过渡状态） |
| 已完结 | COMPLETED | 采收完成，批次结束 |
| 已废弃 | ABANDONED | 因疫病、灾害等原因废弃 |

正常推进路径：已计划 → 已播种 → 生长期 → 待采收 → 采收中 → 已完结

任何非终态均可转为「已废弃」。

### 4.3 操作步骤详解

#### 第一步：登录系统

使用任一账户登录，不同角色会看到不同菜单。系统同时在后端拦截路由，不能仅靠手动输入 URL 绕过权限。

#### 第二步：查看仪表盘

首页展示地块可视化布局（色块图）、批次状态统计（柱状图）、营收数据。鼠标悬停地块可查看批次详情。

#### 第三步：管理种植批次

进入「种植批次」页面：
1. 新增批次：填写批次号，选择地块和作物，填写计划面积、播种日期和预计采收日期
2. 新批次默认进入「已计划」状态
3. 查看当前所有批次及其状态
4. 选择目标状态，点击「推进」按钮控制批次生命周期
5. 状态推进记录自动写入 `batch_status_log` 表

状态名称均为中文："已计划"→"已播种"→"生长期"→"待采收"→"采收中"→"已完结"。

#### 第四步：记录农事作业

进入「农事作业」页面：
1. 选择目标批次
2. 选择作业类型：播种、施肥、施药、灌溉、除草、病虫害巡检、采收
3. 填写作业日期、工人姓名
4. 选择消耗的农资（可选），填写消耗数量
5. 填写作业说明
6. 提交后，系统自动扣减对应农资库存

**重要特性**：农事作业记录不可篡改（数据库触发器保护）。如需修正，应新建一条「红冲修正」类型的作业进行冲销。

#### 第五步：安全采收

采收前系统自动检查安全间隔期：
- 最近一次施药日期到采收日期必须满足该农药的 `safe_interval_days`（安全间隔天数）
- 如施用了吡虫啉（安全间隔 7 天），则施药后 7 天内不能采收
- 不满足间隔期会阻止采收并给出提示

采收成功后：
- 批次状态推进到「已完结」
- 自动生成唯一溯源码（格式：F000001-YYYYMMDD-批次ID）
- 写入采收记录（harvest_record 表）

说明：如果批次已经被人工推进到「采收中」，仍可在采收页面登记采收，系统会继续完成「采收中 → 已完结」并生成溯源码。已完结批次不能重复提交采收，避免重复追溯码和重复采收记录。

#### 第六步：溯源查询

每个采收成功的批次都有独立的溯源页面，访问：
```
http://localhost:8080/trace/{溯源码}
```

溯源页面公开显示：
- 作物名称、品种
- 产地地块、土壤类型
- 采收日期、采收量
- 脱敏农事时间线（操作类型、日期、人员，物资品牌和成本默认隐藏）
- 溯源二维码（可扫码查看）

无需登录即可访问溯源页面。

溯源二维码会自动使用手机可访问的地址：如果 PC 端页面是通过 `localhost` 打开的，系统会优先探测本机局域网 IP 并生成类似 `http://192.168.x.x:8080/trace/{溯源码}` 的二维码。生产环境可通过 `fpms.trace.base-url` 配置固定公网或内网访问地址。

#### 第七步：客户购买在售农作物

客户登录后默认进入「购买农作物」页面：
1. 查看启用且设置了参考售价的在售农作物
2. 输入购买数量，点击「立即购买」
3. 系统按当前作物参考售价计算金额
4. 成功购买后写入 `sale_order` 销售流水，备注为「客户自助购买」
5. 客户可在页面下方或「销售记录」中查看本人购买流水

说明：客户不能手工填写客户名、产品名或单价，避免越权伪造销售记录。

#### 第八步：销售管理

进入「销售管理」页面：
1. 填写客户名称、产品名称
2. 输入销售数量和单价
3. 系统自动计算总金额
4. 仪表盘营收 KPI 自动汇总当月和全部销售额

## 五、农资库存管理

### 5.1 库存操作

仓库管理员可以在库存管理页面：
- 查看所有农资及其当前库存量
- 新增种子、化肥、农药等农资，并维护采购单价
- 新增种子需要绑定的可种植作物
- 执行入库操作（增加库存）
- 查看出库流水（农事作业自动生成的出库记录）

### 5.2 自动扣减

每次农事作业消耗农资时，系统自动：
1. 检查库存是否充足（扣减数量 ≤ 可用库存）
2. 扣减 `stock_inventory.quantity`
3. 生成 `stock_out_order` 出库流水
4. 记录 `plant_operation_material` 关联信息

## 九、常见问题

### Q: 为什么采收按钮点不了？
A: 检查两点：(1) 批次状态是否为「待采收」或「采收中」，如果不是需要先推进状态；(2) 最近一次施药是否满足安全间隔期，不满足会被系统阻止。

### Q: 农事作业能修改或删除吗？
A: 不能。农事作业受数据库触发器保护，禁止 UPDATE 和 DELETE。如需纠正，请新增一条「红冲修正」类型的作业记录。

### Q: 溯源码在哪里生成？
A: 采收时会自动生成溯源码并写入数据库。生成逻辑在 `HarvestService.generateTraceCode()`，格式为 `F000001-YYYYMMDD-批次ID(36进制)`。

### Q: 溯源信息哪些是公开的？
A: 可通过 `trace_public_field` 表配置，默认公开：作物名称、地块名称、采收日期、农事时间线。物料品牌和成本信息默认隐藏。
